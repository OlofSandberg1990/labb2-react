    Numbers game

    Detta consoleapp-program, skrivet i Visual Studio Code, är ett enkelt nummergissningsspel där användaren ska försöka gissa vilket nummer datorn "tänker" på. Programmet startar med att datorn slumpmässigt väljer ett nummer inom ett fördefinierat intervall, till exempel mellan 1 och 100. Användaren informeras om att de har 5 försök på sig att gissa rätt nummer. Vid varje gissning ger programmet feedback till användaren om gissningen var för hög, för låg, eller korrekt. Om användaren gissar rätt nummer inom de tillåtna försöken, gratulerar programmet användaren och avslutar spelet. Om användaren inte lyckas gissa rätt nummer inom de 5 försöken, meddelar programmet vad det korrekta numret var och avslutar sedan spelet. Programmet hanterar även felaktiga inmatningar från användaren, till exempel om de matar in icke-numeriska värden eller nummer utanför det tillåtna intervallet, och instruerar användaren att mata in ett giltigt nummer. Detta spel är en utmärkt övning för nybörjare i programmering för att lära sig grundläggande koncept som loopar, villkorssatser, användarinteraktion från konsolen och slumpmässig nummergenerering.

    AtM machine

    Programmet Bankomat2.0 är ett konsolbaserat bankapplikationsprogram som simulerar grundläggande banktjänster. Användare kan logga in i systemet genom att ange sitt användarnamn och en 4-siffrig pinkod. Efter inloggning visas en meny med olika alternativ, inklusive att visa konton och saldo, överföring mellan konton, insättning, uttag, och möjlighet att logga ut. Varje användare har en lista över bankkonton (AnvändarkontonList), där varje konto (Konton) har ett unikt namn och ett saldo. Programmet tillåter användare att utföra transaktioner såsom att sätta in och ta ut pengar, samt överföra pengar mellan sina egna konton. Användaruppgifter och kontoinformation lagras i ett dictionary (kundregisterDictionary) för snabb åtkomst och hantering. Felhantering är implementerad för att säkerställa korrekt inmatning av data, till exempel att pinkoden är 4 siffror och att insatta eller uttagna belopp är giltiga decimaltal. Om användaren misslyckas med att ange rätt pinkod tre gånger i rad avslutas programmet som en säkerhetsåtgärd. Det övergripande syftet med Bankomat2.0 är att ge en enkel, men funktionell, simulering av vanliga bankoperationer i en konsolmiljö, med användarvänliga meddelanden och instruktioner för att guida användaren genom olika banktjänster.

    Database project

    Det här projektet utvecklat i Visual Studio, använder Entity Framework för att hantera data med en Code First-ansats, snarare än Database First, och SQL Server Management Studio (SSMS) för direkt databashantering. Programmet inkluderar en konsolbaserad meny som möjliggör åtkomst till och hantering av olika datatyper som skolan efterfrågar. Skoladministratörer kan hantera personaldata, inklusive namn, befattningar och anställningstid, samt lägga till ny personal. Elevers information, såsom vilken klass de tillhör och deras betyg i kurser, lagras också, tillsammans med information om lärare som tilldelat betygen och datumen för dessa betyg. Projektet möjliggör även frågor om antalet lärare per avdelning, elevinformation, och aktiva kurser via Entity Framework. Vidare tillåter det detaljerad ekonomisk analys genom SQL-frågor, inklusive total löneutbetalning per avdelning och genomsnittslöner. Dessutom inkluderar systemet funktionalitet för att skapa Stored Procedures för att hämta specifik elevinformation med hjälp av ID och genomföra betygsättningar med transaktioner för att hantera fel säkert.

    Bank

    Code-crusader Bank är ett gruppprojekt utvecklat i C# med .NET, som simulerar ett enkelt banksystem. Programmet består av sjutton olika klasser som hanterar olika aspekter av bankverksamheten, inklusive kontohantering, lån, valutakurser och användaradministration. Användare kan logga in antingen som administratör eller som vanlig användare, där varje roll har tillgång till specifika funktioner. Administratören kan skapa nya användare och ställa in valutakurser, medan användare kan öppna sparkonton och lönekonto, genomföra överföringar mellan konton, begära lån baserat på deras saldo, och granska sin överföringshistorik. Det finns även en funktion för att överföra pengar till andra användare och skapa överföringsloggar för att spåra transaktionshistorik. Programmet inleds med en inloggningsskärm som ger användarna tre försök att logga in, med risken att applikationen stängs vid upprepade misslyckanden. Detta projekt erbjuder en omfattande simulering av bankverksamhet med fokus på användarinteraktion och administrativa funktioner.
